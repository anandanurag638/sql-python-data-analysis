# sql-python-data-analysis
created a detailed repo for organizing my code and analysis for my data analytics journey.
**Inner Joins:** Used to fetch records with matching values in both tables (e.g., matching employees to their specific salary records).
* **Right Outer Joins:** Implemented to identify all salary records, including cases where demographic data might be missing (identifying NULLs).
** left join ** used left join on employee demographics and employee salary table.
* **Self-Joins:** Applied logic to compare rows within the same table—useful for hierarchical data or sequential logic (like the `ID + 1` challenge).
* **Multiple Table Joins:** Successfully chained three tables (`Demographics` → `Salary` → `Departments`) to create a complete 360-degree view of the organization.
* **Aliasing:** Used `dem`, `sal`, and `dept` to write clean, readable, and professional-grade SQL code.
